# Guia de Tono y Voz - Pedily

> UX Writing para el mercado peruano con vision de expansion a Latam y USA.

**Version:** 1.0
**Fecha:** Febrero 2026

---

## 1. Resumen Ejecutivo

Pedily es una plataforma SaaS B2B para negocios de comida. Nuestro usuario principal es el **dueno o administrador de restaurante peruano** (25-55 anos), que busca una solucion tecnologica pero valora la cercania y la confianza.

### Principios de Comunicacion

| Principio | Descripcion |
|-----------|-------------|
| **Cercano, no coloquial** | Usamos "tu" pero evitamos jerga excesiva |
| **Claro, no simplista** | Explicamos sin condescendencia |
| **Confiable, no frio** | Profesional pero con calidez humana |
| **Directo, no brusco** | Vamos al punto pero con tacto |

---

## 2. Forma de Tratamiento: Tu vs Usted

### Recomendacion: **Tuteo consistente**

Para Pedily, recomendamos usar **"tu"** en toda la comunicacion por las siguientes razones:

1. **Mercado objetivo**: Duenos de restaurantes jovenes y emprendedores digitales
2. **Contexto digital**: Apps y SaaS en Peru usan tuteo (Yape, Rappi, etc.)
3. **Posicionamiento**: Marca moderna, accesible, no corporativa
4. **Solidaridad**: "Tu" crea sensacion de "somos uno de los tuyos"

### Excepciones para Usted

- Comunicaciones legales (Terminos y Condiciones, Politica de Privacidad)
- Respuestas a quejas formales de clientes
- Comunicacion con instituciones (SUNAT, bancos)

### Ejemplos Comparativos

| Contexto | Con "Tu" (Recomendado) | Con "Usted" (Evitar) |
|----------|------------------------|---------------------|
| Bienvenida | "Bienvenido a Pedily, tu plataforma de pedidos" | "Bienvenido a Pedily, su plataforma de pedidos" |
| Error | "Algo salio mal. Intenta de nuevo." | "Ha ocurrido un error. Intente nuevamente." |
| CTA | "Empieza tu prueba gratis" | "Inicie su prueba gratuita" |
| Confirmacion | "Tu pedido fue recibido" | "Su pedido ha sido recibido" |

## 3. Atributos del Tono de Voz

### 3.1 Cercano pero Profesional

Pedily habla como un **colega experto** que te ayuda, no como un manual tecnico ni como tu pata del barrio.

```
Mal:  "El sistema ha procesado satisfactoriamente su solicitud de registro."
Mal:  "Ya estas adentro causa, que bacan!"

Bien: "Listo, tu cuenta esta creada. Ahora configura tu primera tienda."
```

### 3.2 Claro y Conciso

Los usuarios escanean, no leen. Cada palabra debe ganarse su lugar.

```
Mal:  "Para poder continuar con el proceso de configuracion de su tienda
       en linea, es necesario que complete los siguientes campos requeridos."

Bien: "Completa estos datos para crear tu tienda."
```

### 3.3 Empoderador

Hacemos que el usuario se sienta capaz, no perdido.

```
Mal:  "Error: Configuracion incompleta. Contacte soporte."

Bien: "Falta configurar tu metodo de pago. Es rapido, solo toma 2 minutos."
```

### 3.4 Honesto y Transparente

Los peruanos valoran la honestidad. Si algo falla, lo decimos claro.

```
Mal:  "Estamos experimentando dificultades tecnicas temporales."

Bien: "Nuestro servidor esta caido. Estamos trabajando en ello."
```

## 4. Peruanismos Estrategicos

### 4.1 Palabras que SI usamos (con moderacion)

Estas palabras son ampliamente entendidas y dan identidad local sin ser excluyentes:

| Palabra | Significado | Uso en Pedily |
|---------|-------------|---------------|
| **Listo** | Ok, hecho | "Listo, tu pedido fue enviado" |
| **Genial** | Excelente | "Genial, ya tienes tu primera venta" |
| **Al toque** | Inmediatamente | "Recibe notificaciones al toque" |
| **Chamba** | Trabajo | Solo en contexto muy casual (blog, redes) |

### 4.2 Palabras que NO usamos en la interfaz

Evitamos jerga que pueda:
- Excluir a usuarios mayores o de otras regiones
- Sonar poco profesional para un producto B2B
- No ser entendida si expandimos a otros paises

| Evitar | Alternativa |
|--------|-------------|
| Bacan | Genial, excelente |
| Pata/Causa | Cliente, usuario |
| Jato | Local, tienda |
| Jamear | Comer (usar "pedido de comida") |
| Pituco | (no usar) |

### 4.3 Diminutivos

El espanol peruano usa diminutivos para suavizar. Usarlos **con moderacion**:

```
Aceptable: "Un momentito mientras procesamos tu pago"
Evitar:    "Agregadito al carrito" (excesivo)
```

## 5. Estructura de Mensajes

### 5.1 Mensajes de Exito

**Formula**: Confirmacion + Que sigue

```
"Pedido confirmado. Tu cliente recibira una notificacion."
"Producto agregado. Puedes seguir editando tu catalogo."
"Pago recibido. El pedido pasa a cocina automaticamente."
```

### 5.2 Mensajes de Error

**Formula**: Que paso + Como solucionarlo

Los peruanos evitan el conflicto directo. Nunca culpes al usuario.

```
Mal:  "Error: Ingresaste un email invalido."
Bien: "El email no parece correcto. Revisa si tiene @ y un dominio."

Mal:  "No tienes permisos para esta accion."
Bien: "Esta accion requiere permisos de administrador. Pide acceso a tu admin."
```

### 5.3 Estados Vacios

**Formula**: Explica el estado + Invita a la accion

```
"Aun no tienes pedidos hoy. Cuando llegue uno, aparecera aqui."
"Tu catalogo esta vacio. Agrega tu primer producto para empezar a vender."
```

### 5.4 Llamados a Accion (CTAs)

**Formula**: Verbo en infinitivo o imperativo + Objeto claro

| Contexto | CTA Recomendado |
|----------|-----------------|
| Registro | "Crear cuenta gratis" |
| Inicio de prueba | "Empezar prueba de 14 dias" |
| Agregar producto | "Agregar producto" |
| Guardar cambios | "Guardar cambios" |
| Ver detalles | "Ver detalles" |
| Contacto | "Hablar con soporte" |

**Evitar CTAs vagos**: "Enviar", "Continuar", "Aceptar" (sin contexto)

## 6. Comparativa Regional

### 6.1 Peru (Mercado Principal)

| Aspecto | Recomendacion Peru |
|---------|-------------------|
| Tratamiento | Tu (tuteo) |
| Formalidad | Media-baja, cercana |
| Expresividad | Alta, calidez |
| Jerga | Minima, selectiva |
| Diminutivos | Moderados |
| Comunicacion | Indirecta para errores |

### 6.2 Argentina (Expansion Futura)

| Aspecto | Adaptacion Argentina |
|---------|---------------------|
| Tratamiento | **Vos** (voseo rioplatense) |
| Conjugacion | "Crea tu cuenta" → "Crea tu cuenta" (igual) pero "Tienes" → "Tenes" |
| Expresiones | "Genial" → "Barbaro", "Copado" |
| Tono | Mas directo, menos formal |
| Jerga | "Laburo" (trabajo), "Guita" (dinero) - solo en marketing |

**Ejemplo de adaptacion:**
```
Peru:      "Listo, tu pedido fue confirmado."
Argentina: "Listo, tu pedido fue confirmado." (sin cambios en UI)
           "Barbaro, ya tenes todo configurado." (en marketing)
```

### 6.3 Chile (Expansion Futura)

| Aspecto | Adaptacion Chile |
|---------|-----------------|
| Tratamiento | Tu (tuteo) |
| Formalidad | Similar a Peru |
| Expresiones | "Bacan" tambien se usa, "Cachai" (entiendes) |
| Tono | Puede ser ligeramente mas informal |
| Velocidad | Chilenos hablan rapido, copy puede ser mas corto |

**Ejemplo de adaptacion:**
```
Peru:  "Configura tu tienda en minutos."
Chile: "Configura tu tienda en minutos." (sin cambios)
```

### 6.4 USA - Espanol Neutro (Expansion Futura)

| Aspecto | Adaptacion USA/Neutro |
|---------|----------------------|
| Tratamiento | Tu (tuteo estandar) |
| Formalidad | Media |
| Jerga | Ninguna regional |
| Anglicismos | Mas aceptados ("delivery", "feedback") |
| Audiencia | Latinos de multiples origenes |

**Ejemplo de adaptacion:**
```
Peru:  "Recibe pedidos de delivery y recojo en tienda."
USA:   "Recibe pedidos de delivery y pickup." (anglicismo aceptado)
```

### 6.5 USA - Ingles

| Aspecto | Recomendacion Ingles |
|---------|---------------------|
| Tratamiento | "You" (sin distincion formal) |
| Tono | Friendly, professional |
| Estilo | Concise, action-oriented |
| Voz | Active voice preferred |

**Ejemplo:**
```
Espanol: "Tu pedido fue confirmado."
English: "Your order is confirmed."
         o mejor: "Order confirmed." (mas conciso)
```

## 7. Glosario de Terminos Pedily

Terminos consistentes en toda la plataforma:

| Termino Pedily | Definicion | NO usar |
|----------------|------------|---------|
| **Tienda** | El negocio del cliente | Local, restaurante, establecimiento |
| **Catalogo** | Lista de productos | Menu, carta (solo en contexto de comida) |
| **Producto** | Item del catalogo | Articulo, plato |
| **Pedido** | Orden de un cliente | Orden (anglicismo), solicitud |
| **Cliente** | Quien hace el pedido | Comensal, usuario final |
| **Hub** | Panel de gestion de pedidos | Dashboard (en UI en espanol) |
| **Cocina** | Vista para preparacion | Back of house |
| **Delivery** | Entrega a domicilio | Envio, despacho (mantener "delivery") |
| **Recojo en tienda** | Cliente recoge | Pickup, take away |
| **Kiosko** | Terminal de autoservicio | Totem, POS (en contexto de cliente) |

## 8. Ejemplos de UX Writing por Seccion

### 8.1 Onboarding

```
Pantalla 1:
Titulo:    "Bienvenido a Pedily"
Subtitulo: "Vamos a crear tu tienda en 3 pasos."
CTA:       "Empezar"

Pantalla 2:
Titulo:    "Sobre tu negocio"
Campo:     "Nombre de tu tienda"
Ayuda:     "Es el nombre que veran tus clientes."

Pantalla 3:
Titulo:    "Ya casi esta"
Subtitulo: "Agrega tu primer producto para empezar a recibir pedidos."
CTA:       "Agregar producto"
Skip:      "Hacerlo despues"
```

### 8.2 Hub de Pedidos

```
Nuevo pedido:
Notificacion: "Nuevo pedido de Juan - S/ 45.00"
Accion:       "Ver pedido"

Pedido listo:
Boton:        "Marcar como listo"
Confirmacion: "Pedido marcado como listo. El cliente fue notificado."

Sin pedidos:
Estado vacio: "No hay pedidos pendientes. Cuando llegue uno, lo veras aqui."
```

### 8.3 Errores y Validaciones

```
Campo vacio:
"El nombre es obligatorio."

Email invalido:
"Ingresa un email valido (ejemplo: tu@correo.com)"

Pago fallido:
"El pago no pudo procesarse. Revisa los datos de tu tarjeta o intenta con otro metodo."

Sin conexion:
"Sin conexion a internet. Los cambios se guardaran cuando vuelvas a conectarte."

Sesion expirada:
"Tu sesion expiro por seguridad. Inicia sesion de nuevo."
```

### 8.4 Confirmaciones

```
Producto creado:
"Producto agregado a tu catalogo."

Configuracion guardada:
"Cambios guardados."

Cuenta eliminada (confirmacion):
Titulo:    "Eliminar cuenta?"
Mensaje:   "Se eliminaran todos tus datos, pedidos e historial. Esta accion no se puede deshacer."
CTA:       "Si, eliminar cuenta"
Cancel:    "Cancelar"
```

## 9. Checklist de Revision

Antes de publicar cualquier texto, verificar:

- [ ] Usa "tu" consistentemente (excepto documentos legales)
- [ ] Es claro en primera lectura
- [ ] Tiene menos de 15 palabras (para mensajes cortos)
- [ ] Usa verbos activos, no pasivos
- [ ] Evita jerga tecnica innecesaria
- [ ] No culpa al usuario en errores
- [ ] Incluye siguiente paso o accion clara
- [ ] Es consistente con el glosario de terminos
- [ ] Funciona si se traduce literalmente (para expansion)
